1.以solidworks繪製base和link長度分別為1.13 2.5 3 m
   再組合圖中組合並調整初始角度為13
   轉存為stl檔

2.匯入blender調整link1起始點歸零加入外觀顏色並以obj匯出

   link1 device啟用rotationalmotor

相關問題

  1. WorldInfo 節點的作用為何？
    WorldInfo 節點提供場景全域設定與元資料。

  2. Viewpoint 節點設計時需考慮哪些參數？
    需考慮 position、orientation 與 fieldOfView 等參數。

  3. TexturedBackground 與 TexturedBackgroundLight 在場景中的功能？
    TexturedBackground 負責背景影像，TexturedBackgroundLight 控制背景光線效果。

  4. 為什麼需要使用 EXTERNPROTO 來引用背景相關的 PROTO？
    可重用外部定義的 PROTO 節點，保持場景簡潔。

  5. 場景中為何有兩個 BASE（BASE 與 BASE2）？
    可能區分不同底座或不同功能的基座結構。

  6. CadShape 節點如何使用，其作用與限制為何？
    用於導入 CAD 模型，限制在於檔案格式與渲染效能。

  7. Mesh 作為 boundingObject 有什麼效果？
    能更精確定義碰撞範圍，提高物理模擬精度。

  8. 資料夾路徑 "../split_parts/" 的用途為何？
    存放分割零件的外部模型檔案。

  9. Robot 節點的結構和功能？
    包含機器人所有組件，管理控制與物理屬性。

  10. HingeJoint 節點在本結構中如何應用？
    用於連接可旋轉的連桿，實現關節運動。

  11. HingeJointParameters 的 axis 與 anchor 參數說明？
    axis 定義旋轉軸方向，anchor 定義旋轉中心位置。

  12. RotationalMotor 的功能是什麼，如何與關節對應？
    控制關節旋轉速度與力矩，連接對應 HingeJoint。

  13. SolidReference 的用途與設計意義？
    用於引用其他 Solid，實現多連桿連結。

  14. 為什麼每個連桿（link）都需要 physics 與 boundingObject？
    提供物理屬性與碰撞邊界，確保模擬真實性。

  15. name 屬性在 Solid 與 Robot 節點中的差異？
    Solid 的 name 表示零件名稱，Robot 的 name 表示整體機器人。

  16. JOINT1~JOINT4 分別對應機構的哪個部位？
    對應機構中的不同旋轉關節位置。

  17. anchor 參數對機構運動的實際影響？
    決定關節旋轉的固定點位置與運動軌跡。

  18. DEF 與 USE 的差異與用途？
    DEF 定義節點，USE 重複使用已定義節點。

  19. 場景中 LINK1、LINK2、LINK3 的結構意義？
    表示機構中的不同連桿層級與連結。

  20. 為何要將外部 CAD 檔案（obj）導入 Webots？
    提高模型細節與真實感。

  21. 如何在 Webots 中檢查導入 CAD 物件的碰撞情形？
    使用碰撞檢測工具與 boundingObject 設定。

  22. controller 屬性如何讓機器人執行指定程式？
    指定控制程式的名稱，啟動對應控制邏輯。

  23. 如果要更改關節旋轉範圍，該修改哪些參數？
    修改 HingeJointParameters 中的範圍限制參數。

  24. 如何確保多個 link 結構組裝時不產生重疊？
    調整 translation 與 rotation，並使用 Pose 精確定位。

  25. 若要讓 LINK3 可由 JOINT3 控制，需注意哪些設定？
    確保 JOINT3 關聯到 LINK3 並配置正確馬達。

  26. 機構結構層級在 VRML/WBT 檔案中的重要性？
    影響繼承關係與物理模擬的正確性。

  27. 若 CAD 檔案位置錯誤或缺失，Webots 會怎麼處理？
    模型無法載入並顯示錯誤警告。

  28. 如何於場景內為各零件添加材質或調整外觀？
    修改 Appearance 節點的材質屬性如 diffuseColor。

  29. Viewpoint 的 orientation 和 position 如何影響觀察？
    position 決定視角位置，orientation 決定視角方向。

  30. 製作此一場景時，需注意哪些命名與結構規範？
    保持命名一致性與層級清晰，方便管理與調試。
